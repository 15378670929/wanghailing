表单

在创建表单时，我们一般会将表单写为受控组件，因为表单具有HTML表单行为，即在用户提交表单后浏览到新页面。如果你在 React 中执行相同的代码，它依然有效。但大多数情况下，使用 JavaScript 函数可以很方便的处理表单的提交， 同时还可以访问用户填写的表单数据。实现这种效果的标准方式是使用“受控组件”。

将表单写为受控组件需要注意2点：

表单的value是state里面的值，将state写成表单的唯一数据源；

表单绑定onChang事件，并在onChange触发时，改变state里对应表单value的值。

完成这2点，一个表单元素就改成了受控组件

状态提升

react里的数据源最好只有一个相对应的唯一数据源，通常将state先添加到渲染数据的组件中，如果其他组件也需要这个state里的数据，那么将这个数据放在他们最近共同的父组件中，这就是状态提升。
